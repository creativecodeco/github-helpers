GitHub Helpers

Guía de Seguridad y Gestión de Tokens

Aprende cómo gestionar de manera segura la visualización de tus repositorios privados.

¿Cómo se almacenan y protegen mis datos?

Tu privacidad y seguridad son nuestra máxima prioridad. Cuando decides registrar tu token personal en GitHub Helpers:

  • Cifrado Simétrico Fuerte: Tu token se cifra inmediatamente en el servidor utilizando el algoritmo estándar de la industria AES-256-GCM.
  • Clave Secreta Exclusiva: La llave de descifrado reside únicamente en las variables de entorno de producción del servidor. Nadie (ni siquiera los administradores de la base de datos) puede ver tu token sin esa clave.
  • Auditoría de Consentimiento: Registramos que has aceptado explícitamente el almacenamiento junto con un timestamp y una firma basada en la dirección IP/User-Agent en el momento del registro.

Paso a Paso: Crear un Token Seguro y de Solo Lectura

Por seguridad, no aceptamos tokens con permisos de escritura o administración. Sigue estos pasos para crear el token correcto:

  1. Ve a tu cuenta de GitHub, haz clic en tu foto de perfil arriba a la derecha y entra a Settings.
  2. En el menú izquierdo, baja hasta el final y haz clic en Developer Settings.
  3. Selecciona Personal Access Tokens y haz clic en Fine-grained tokens (o accede directamente a través de este enlace directo de GitHub).
  4. Presiona el botón Generate new token.
  5. Configura los siguientes campos obligatorios:
    • Token name: Pon un nombre descriptivo (ej: github-helpers-stats).
    • Repository access: Elige All repositories para ver todas tus estadísticas privadas o Only select repositories si solo deseas incluir ciertos repositorios.
    • Permissions: En el menú desplegable de Repository permissions, busca Metadata y cámbialo a Access: Read-only. Asegúrate de no otorgar ningún otro permiso.
  6. Haz clic en Generate token al final de la página y copia el token generado.
Atención: Si intentas subir un token con permisos de escritura (como el scope general repo de un Classic Token), nuestro servidor rechazará el registro automáticamente y no guardará ningún dato.

¿Cómo puedo revocar o eliminar mi token almacenado?

Tienes control absoluto sobre tus datos en todo momento. Ofrecemos tres métodos para revocar el acceso de tu token:

1 Revocación rápida desde la Web

Si estás usando el mismo navegador desde el cual registraste el token, simplemente haz clic en el botón Revocar Token en la interfaz. El token se borrará de la base de datos inmediatamente.

2 Revocación si has extraviado el token original

Si has perdido el token original o estás en un navegador diferente y quieres borrar tus datos de nuestros servidores:

  1. Genera un token de GitHub temporal y rápido (incluso sin ningún permiso).
  2. En la sección "Eliminar Token" del panel, introduce tu nombre de usuario y el token temporal.
  3. El backend consultará a GitHub a quién pertenece ese token temporal para comprobar tu identidad. Si el dueño del token coincide con tu nombre de usuario, el servidor eliminará tu token original cifrado de nuestra base de datos.
  4. Desecha y borra el token temporal en GitHub.

3 Desactivación inmediata desde GitHub

Puedes invalidar el token inmediatamente de forma remota. Ve a tu lista de tokens en GitHub y haz clic en Revoke o Delete al lado del token. A partir de ese milisegundo, cualquier llamada que nuestro servidor intente hacer con él fallará por parte de GitHub y tus estadísticas privadas dejarán de mostrarse automáticamente.
